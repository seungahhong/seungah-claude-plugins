# 실행 환경 카탈로그 (Execution Environments)

메타 프롬프팅의 마지막 단계는 **"실행 환경에 맞게 변환하기"** 다. 같은 컨텍스트라도 **어느 도구에 주입할 프롬프트인지에 따라 다르게 깎아야** 한다. 이 카탈로그는 환경별로 (1) 하드 제약, (2) 반드시 넣어야 할 변수, (3) 엔딩/중지 조건을 정리한다. `prompt-carver`가 이 표를 참조해 최종 프롬프트를 환경에 최적화한다.

> **정직성**: 아래 수치·제약은 도구가 자주 바꾼다. 하드 제약(길이 제한 등)은 **깎기 직전에 사용자에게 실제 값을 확인**하고, 확인 불가하면 프롬프트에 "이 환경의 길이 제한 내로 맞춰라"처럼 **상대적 지시**로 넣는다. 특정 숫자를 사실로 단정하지 않는다.

---

## 환경별 요약

| 환경 | 하드 제약 | 반드시 넣을 변수 | 엔딩/중지 조건 |
|------|-----------|-----------------|----------------|
| **Goal / loop (자율 반복)** | 길이 상한(예: ~4000자 수준, 도구별 상이·확인 필요) | 목표, 검증 가능한 완료 정의, 반복 범위 | **명시적 중지 조건 필수** — 없으면 무한 반복. "무엇이 참이면 멈춘다" |
| **ultracode / 워크플로우 오케스트레이션** | (동적 하네스) 토큰 예산 | 제약 조건, 병렬화 스코프, 검증/집계 방식 | 예산 소진·수렴 기준·에이전트 수 상한 |
| **Claude Code / 코딩 에이전트 (긴 작업)** | 컨텍스트 예산 | 대상 파일/모듈 경계, 실행·검증 방법, 가역/위험 분류 | 테스트/센서 통과, "완료" 판정 기준 |
| **이미지 생성** | 프롬프트 길이·토큰 | 구도, 피사체, 스타일, 조명, 카메라(렌즈/앵글) | 결과물의 명확한 채택 기준(무엇이 있으면 성공) |
| **리서치 / 조사** | (환경별) | 출처 기준, 조사 범위, 검증 방식, 산출 형식 | 커버리지 기준·검증 통과·인용 요건 |
| **일반 채팅 (단발 산출)** | 응답 길이·컨텍스트 | 역할, 산출 형식, 예시, 성공 기준 | 산출 형식 충족 여부 |

## 환경 상세

### Goal / loop (자율 반복 프롬프트)
- **왜 특별한가**: 며칠씩 도는 긴 작업. 처음 프롬프트가 전체 결과를 좌우한다. 스티어링이 가능해도, **초기 프롬프트의 명확성**이 시간·토큰 낭비를 결정한다.
- **중지 조건이 핵심**: "언제 멈춰야 하는지"를 안 넣으면 계속 돈다. 검증 가능한 완료 정의(무엇이 참이면 끝)를 반드시 포함.
- **길이 상한**: 상한이 있는 도구가 많다(값은 도구별·시점별로 다르므로 확인). 상한이 있으면 **큰 그림을 먼저 그린 뒤 상한 내로 축소**한다(carving-principles.md의 expansion→compression).

### ultracode / 워크플로우 오케스트레이션
- **동적 하네스**: 실행 중 팬아웃·검증 단계가 늘어난다. **제약 조건**(무엇을 하면 안 되는지)과 **검증/집계 방식**을 명시하면 결과가 안정적이다.
- 병렬화 스코프·토큰 예산·수렴 기준을 프롬프트에 담는다.

### Claude Code / 코딩 에이전트
- **파일/모듈 경계**와 **실행·검증 방법**(어떻게 동작을 확인하는지)을 명시. "테스트 통과 = 동작 보존"이 아님을 감안한 검증 지시.
- 가역/위험 분류(안전 임계 변경은 사람 게이트)를 프롬프트 수준에서 언급하면 안전.

### 이미지 생성
- 결과물 품질을 가르는 변수: **구도(composition) · 피사체(subject) · 스타일(style) · 조명(lighting) · 카메라(camera: 렌즈/앵글/거리)**.
- 이 변수들이 곧 "엔딩 조건"에 해당한다 — 무엇이 프레임 안에 어떻게 있어야 성공인지.

### 리서치 / 조사
- **출처 기준**(어떤 소스가 신뢰 가능), **조사 범위**(무엇을 포함/제외), **검증 방식**(교차검증·적대 검증), **산출 형식**(표·인용).
- 이런 요소조차 사용자가 직접 쓸 필요는 없다 — "이 리서치 환경에 맞게 출처·범위·검증을 갖춰 깎아 달라"고 요청하면 된다.

### 일반 채팅 (단발 산출)
- 역할·산출 형식·예시(few-shot)·성공 기준. 가장 단순하지만, **성공 기준을 명시**하면 완성도가 오른다.

## 사용 규약 (prompt-carver가 따른다)

1. **환경을 먼저 확정**한다(Phase 0에서). 미정이면 위 표로 선택을 유도한다.
2. 해당 환경의 **하드 제약 실제값을 사용자에게 확인**한다(모르면 상대적 지시로 대체).
3. **반드시 넣을 변수**가 프롬프트에 다 들어갔는지 점검한다.
4. **엔딩/중지 조건**을 빠뜨리지 않는다(특히 Goal/loop).
5. 이 카탈로그에 없는 환경이면, 같은 3요소(하드 제약·필수 변수·엔딩 조건)를 사용자와 함께 도출해 적용한다.
