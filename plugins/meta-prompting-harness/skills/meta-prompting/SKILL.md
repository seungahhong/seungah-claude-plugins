# Meta-Prompting — 프롬프트를 깎아내는 인터랙티브 오케스트레이터

**메타 프롬프팅**은 "X 만들어줘"라고 직접 지시하는 대신, **X를 만들 좋은 프롬프트를 AI가 최적화해 생성하게 하고, 그 프롬프트를 새 컨텍스트(새 창)에 주입**해 실제 작업을 돌리는 기법이다. 긴 작업(며칠 도는 Goal/loop, 많은 토큰을 태우는 작업)에서 처음부터 명확한 프롬프트를 넣어 시간·토큰 낭비를 막는 게 목적이다.

이 하네스는 널리 쓰이는 메타 프롬프팅 실천 방법론을, 각 기법마다 1차 문헌과 대조해 **근거 등급**을 붙여 재구성했다([references/evidence.md](./references/evidence.md)).

4 Phase로 진행한다: **인테이크·덤핑(Phase 0) → 질문 유도(Phase 1) → 성공·검증 조건(Phase 2) → 환경 맞춤 깎아내기(Phase 3) → 적대 점검·주입 안내(Phase 4)**. 매 Phase 종료 시 핵심 산출물 + 1줄 보고를 제시하고 승인을 받은 뒤에만 다음으로 넘어간다.

## 핵심 프로세스 (3단계)

1. **컨텍스트 덤핑** — 머릿속·회의록·상급자 지시를 정제하지 말고 다 던진다.
2. **프롬프트 깎아내기(carving)** — 질문 유도 → 성공/검증 조건 → 환경 맞춤으로 다듬는다.
3. **점검 후 fresh 컨텍스트에 주입** — 사용자가 도메인 지식으로 수정 → 완성 프롬프트만 새 창에 주입.

## 근거 등급 (정직성 — 모든 Phase가 따른다)

기법을 실증 강도에 따라 정직하게 구분한다. 상세: [references/evidence.md](./references/evidence.md).

- **STRONG(1차 근거·단, 조건부)** — 메타프롬프팅이 직접 프롬프팅 상회(단 모델/과제 조건부·전이 불안정)·생성 전 명확화 질문·성공+검증 조건 명시(Anthropic 공식)·fresh 컨텍스트 주입·은탄환 아님.
- **report-only(근거 약함·folklore)** — 실행 환경 맞춤 변환·큰그림→축소. 실무 관행으로만 제시하고 **"실증됐다"고 말하지 않는다**.
- **REFUTED(미사용)** — "메타프롬프팅=지휘자가 전문가 인스턴스 생성" 프레이밍, "LLM은 스스로 명확화 불가". **쓰지 않는다.**

절대 규칙: 최적화 프롬프트를 **보편 법칙으로 서술하지 않는다**(자기보고·불안정). **전부 위임 금지**. AI가 깎은 프롬프트는 **사용자 도메인 검증 필수**. 이 하네스는 **프롬프트 생성만** 하고 실행(새 창 주입·`/goal`·이미지 생성)은 사용자가 한다. **커밋하지 않는다**.

## 경계 (먼저 읽고 발동 여부를 판단하라)

이 하네스의 고유 판별자는 **다른 도구에 주입할 지시 프롬프트(instruction) 자체를 AI가 생성하게 하고, 질문 유도·성공조건·환경 맞춤으로 깎아내는 것**이다. 다음 작업은 산출물이 프롬프트가 아니므로 범위 밖이다(발동하지 않는다):

- **이미 목표가 있는 상태의 컨텍스트 페이로드 조립·압축·리트리벌** — 산출물이 지시 프롬프트가 아니라 컨텍스트 payload다.
- **Claude 에이전트 하네스/스킬 자체를 생성·진단·개선** — 산출물이 하네스 자산이지 프롬프트가 아니다.
- **기획문서(PRD)·사용자 스토리 작성** — 산출물이 문서다(질문 유도가 겹쳐 보여도 목적이 다르다).
- **계획/방법론 정렬 인터뷰** — 최종 산출물이 프롬프트가 아니면 발동하지 않는다.
- **자율 반복 루프 실행** — 이 하네스는 *그 루프에 넣을 프롬프트를 만들* 뿐 루프를 돌리지 않는다.
- **코드 구현·리뷰·커밋** — 이 하네스는 프롬프트만 만들고 실행하지 않는다.

경계가 모호하면 한 질문으로 확인한다 — "*다른 도구에 넣을 프롬프트*를 다듬으려는 건가요, 아니면 *다른 산출물(컨텍스트 페이로드·문서·하네스·코드)* 을 만들려는 건가요?"

## 참조 문서

- 실행 환경 카탈로그(환경별 하드 제약·필수 변수·엔딩 조건): [references/execution-environments.md](./references/execution-environments.md)
- 깎아내기 원칙(덤핑·질문유도·성공조건·환경맞춤·큰그림→축소·fresh context·도메인 검증·은탄환 아님): [references/carving-principles.md](./references/carving-principles.md)
- 근거 dossier(기법별 학술 검증·등급·1차 소스): [references/evidence.md](./references/evidence.md)

## 산출물 배치

별도 지정이 없으면, **최종 프롬프트는 기본적으로 채팅으로 제시**한다(바로 복사해 새 창에 주입하도록). 사용자가 저장을 원하면 **`.claude/_docs/<슬러그>/`** 에 기록한다(슬러그는 목표명에서 영문 소문자, 시작 시 한 번 확인). 모든 언어는 **한국어**.

```
.claude/_docs/<슬러그>/
  context-dump.md      # Phase 0 덤프 + Phase 1 보강 — 저장 선택 시
  final-prompt.md      # Phase 3 최종 프롬프트(환경 표기 포함) — 저장 선택 시
  critique.md          # Phase 4 점검 리포트 — 저장 선택 시
```

## 내재화 원칙 (모든 Phase)

- **승인 게이트** — 매 Phase 종료 시 핵심 산출물 + 1줄 보고를 제시하고 승인을 받는다. 형식: `[Phase N] <산출 요약> — 다음: <다음 단계>. 진행할까요?`
- **에이전트 호출** — 각 정의는 `../../agents/{name}.md`. **모든 Agent 호출은 `model: "opus"`를 명시**한다.
- **정직성** — 근거 등급을 지키고, report-only를 실증으로 포장하지 않으며, 점수화하지 않는다.

# 인터랙티브 플로우

시작 시 **바로 Phase 0으로 진입**한다. 사용자가 "그냥 프롬프트 만들어줘"로 바로 요청해도, **먼저 목표·실행 환경을 확정하고 컨텍스트 덤핑을 유도**한 뒤 진행한다 — 덤핑 없이 바로 프롬프트를 만들면 사용자 의도가 안 들어가 AI가 임의로 만든 프롬프트 틀에 갇힌다.

> **첫 행동 규약(불변식)** — 사용자가 처음부터 완성된 요청을 줘도, Phase 0의 (1) 실행 환경 확정 + (2) 컨텍스트 덤핑 유도를 건너뛰지 않는다. (사용자가 "덤핑 생략하고 바로 깎아"를 *명시적으로* 요구하면 그때만 축약하되, 전부 위임 경고는 유지한다.)

## Phase 0 — 인테이크 & 컨텍스트 덤핑 (스킬 본문, 대화)

두 가지를 확정하고 덤핑을 유도한다.

1. **무엇을 만들 프롬프트인지** (목표).
2. **어느 실행 환경에 넣을 프롬프트인지** — Goal/loop·ultracode·Claude Code·이미지 생성·리서치·일반 채팅. 미정이면 [references/execution-environments.md](./references/execution-environments.md)로 선택을 유도한다(환경이 깎는 방식을 좌우).
3. **컨텍스트 덤핑 유도**:

> 좋은 프롬프트를 만들려면 먼저 당신 머릿속을 다 꺼내야 합니다. **정제하지 말고** 다 던지세요 — 만들려는 것, 기능, 제약, 회의에서 받은 요청이 있으면 그 메모를 **그대로 복사·붙여넣기** 해도 됩니다.

승인 게이트: `[Phase 0] 목표·실행 환경({환경}) 확정 + 컨텍스트 덤프 수집 — 다음: 질문 유도(Phase 1). 진행할까요?`

## Phase 1 — 질문 유도 (context-elicitor)

`context-elicitor`를 호출해, 덤프에서 **빠진 요소를 AI가 먼저 되묻는** 질문을 생성한다. 근거: 요구 모호성은 정확도를 떨어뜨리고 명확화 질문은 정확도를 올린다(T2, STRONG). 단 명확화는 모델이 자동으로 잘하는 능력이 아니라 **명시 유도**가 필요하다.

```
Agent(
  subagent_type="context-elicitor", model="opus",
  prompt="""
  [역할] 컨텍스트 덤프에서 좋은 프롬프트를 만들기 전에 빠진 요소를 사용자에게 되묻는 질문을 생성한다.
  [입력] 목표: {목표}, 실행 환경: {환경}, 컨텍스트 덤프(원문): {덤프}
  [참조] references/execution-environments.md(환경별 변수), references/evidence.md(T2)
  [규칙] 덤프에 이미 있는 건 재질문 금지. 타겟·핵심기능·레퍼런스·완성도 수준(MVP vs 완성)+환경 변수를 캔다.
         번호 질문 + 각 질문에 필요 이유 1줄, 필수/위임가능 표시. 전부 위임 시도 시 경고(핵심은 사용자가 정해야 함).
  [출력] 질문 목록. 사용자 답변 수집 후 보강된 컨텍스트 요약.
  """
)
```

사용자 답변을 받아 보강 컨텍스트를 정리한다(일부 위임은 정상, 전부 위임은 경고).

승인 게이트: `[Phase 1] 보강 컨텍스트 확정({n}개 질문 해소) — 다음: 성공·검증 조건(Phase 2). 진행할까요?`

## Phase 2 — 성공·검증 조건 명시 (success-definer)

`success-definer`를 호출해 프롬프트에 넣을 specific 성공 조건과 **외부 기준 대비 검증**을 도출한다. 근거: Anthropic 공식 권고(T3, STRONG).

```
Agent(
  subagent_type="success-definer", model="opus",
  prompt="""
  [역할] specific 성공 조건 + 외부 기준 대비 검증 지시를 도출한다.
  [입력] 보강 컨텍스트: {보강}, 목표·환경: {목표/환경}
  [참조] references/execution-environments.md(엔딩 조건), references/evidence.md(T3)
  [규칙] Specific·관측 가능. 검증은 순진한 자기수정 아니라 "[성공 기준]에 비춰 확인" 형태. 완성도 수준 초과 금지.
  [출력] 성공 조건 목록(관측/검증 방법 1줄), 검증 지시 문구, 필수/선택 구분.
  """
)
```

승인 게이트: `[Phase 2] 성공·검증 조건 확정 — 다음: 환경 맞춤 깎아내기(Phase 3). 진행할까요?`

## Phase 3 — 환경 맞춤 깎아내기 (prompt-carver)

`prompt-carver`를 호출해 **실제 실행할 최종 프롬프트**를 만든다. 환경 규약·길이 상한·엔딩 조건을 반영하고 큰 그림→축소로 깎는다.

```
Agent(
  subagent_type="prompt-carver", model="opus",
  prompt="""
  [역할] 대상 환경에 맞게 최종 프롬프트를 깎아낸다.
  [입력] 보강 컨텍스트 + 성공/검증 조건: {…}, 실행 환경: {환경}
  [참조] references/execution-environments.md, references/carving-principles.md, references/evidence.md
  [규칙] 환경 하드 제약·필수 변수·엔딩 조건 반영(Goal은 중지 조건 필수). 길이 상한 있으면 큰그림 먼저→덜 중요한 것 제거로 축소.
         성공/검증 조건을 본문에 녹임. 환경 표기. 환경맞춤·큰그림→축소는 folklore(실증 아님)로 취급. 도메인 사실 지어내기 금지.
  [출력] 최종 프롬프트(복사 가능 완성형), 무엇을 왜 덜어냈는지 요약, 하드 제약 확인 항목.
  """
)
```

승인 게이트: `[Phase 3] 최종 프롬프트 초안({환경}용) — 다음: 적대 점검·주입 안내(Phase 4). 진행할까요?`

## Phase 4 — 적대 점검 + fresh context 주입 안내 (prompt-critic)

`prompt-critic`를 호출해 최종 프롬프트를 적대적으로 점검하고, 사용자 도메인 수정 → fresh 창 주입을 안내한다.

```
Agent(
  subagent_type="prompt-critic", model="opus",
  prompt="""
  [역할] 최종 프롬프트를 적대 점검하고 사용자 수정 + fresh context 주입을 안내한다.
  [입력] 최종 프롬프트: {프롬프트}, 목표·환경·성공조건: {…}
  [참조] references/execution-environments.md, references/evidence.md(T5·T7)
  [규칙] 모호·과대·환경 부적합·검증 누락·미검증 사실·전부위임 잔재 점검(점수화 금지).
```
