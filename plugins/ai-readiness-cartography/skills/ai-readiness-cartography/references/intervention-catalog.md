# 개입 카드 카탈로그 (intervention catalog)

> Phase 2 제안표가 참조하는 **단일 진실 기준**. 코드 **본문** 층(주석·명명·구조)만 다룬다.
> 저장소 층(README·CLAUDE.md·의존 그래프·CI)은 측정 모드(score.py) 소관 — 본문 층 개입과 분리.
> 모든 카드는 **게이트 B(개별 승인)** 후에만 적용한다. 승인 없이 파일을 쓰지 않는다. 커밋은 git-harness 핸드오프.
> 인용은 근거 시트(`references/research/EVIDENCE-SHEET.md`) 밖으로 나가지 않는다.

## C0 · 오도/stale 주석 삭제

- **id**: `C0` · **이름**: 오도·stale 주석 삭제/수정 · **우선순위**: **P1** · **대상 축**: 주석
- **무엇을 바꾸나** — 코드와 어긋난 주석을 제거한다(수정이 아니라 삭제가 기본).
  ```
  // FIXME: 실패 시 3회 재시도 (실제 코드엔 재시도 없음)  ← stale
  send(req)
  ```
  →
  ```
  send(req)
  ```
- **근거** — E-C1 `[PRIMARY-2025+ · 2025-04, NeurIPS 2025]` 오도 주석(MDC) **−13.47pp**(구조 변형 3종 결합 −14.04pp에 필적, CoT로도 방어 안 됨). E-C3 `[PRIMARY-2025+ · 2025-12]` 코드-주석 불일치(CCI)는 실측 가능한 결함 유형. 배경 정의 iComment(2007) `[PRIMARY-pre2025]`.
- **동작 위험**: 없음 — 주석은 실행되지 않는다. **삭제는 동작을 깨뜨릴 수 없다.** · **사실성 위험**: 없음(제거이므로 새 주장을 만들지 않음).
- **1급 센서**: 사람 확인 + 코드 대조(주석 진술 ↔ 실제 제어흐름). · **2급 센서**: 빌드/타입체크가 여전히 통과(`tsc --noEmit` / `go build ./...` / `cargo check`) — 코드는 안 건드렸으므로 불변이어야 정상.
- **적용 도구**: 텍스트 편집(주석 블록 제거). 도구 강제 불필요.
- **롤백**: 해당 hunk `git checkout -- <file>` 또는 revert.
- **게이트**: B.
- **거부 조건**: 주석이 **계약·불변식**을 담고 있고 코드가 그와 다르게 동작하면 → 주석이 아니라 **코드가 버그일 수 있다.** 삭제하지 말고 "코드-주석 불일치, 어느 쪽이 진실인지 판단 필요"로 **사람에게 보고**(C1로도 자동 승격 금지).

## C1 · 계약·불변식 주석 추가/수정

- **id**: `C1` · **이름**: 계약·불변식 주석 추가/수정 · **우선순위**: **P3** · **대상 축**: 주석
- **무엇을 바꾸나** — 코드 대조로 검증된 전제/불변식/에러 조건만 문서화한다.
  ```
  def withdraw(acct, amt): ...
  ```
  →
  ```
  # 계약: amt > 0 이고 acct.balance >= amt (아니면 InsufficientFunds raise)
  # 불변식: 정상 반환 후 acct.balance >= 0
  def withdraw(acct, amt): ...
  ```
- **근거** — E-C2 `[PRIMARY-2025+ · 2025-12, ICSE 2026]` 주석은 **양방향** 인과 인자(품질이 관건)이며 Javadoc/inline/multiline을 별개 개념으로 내재화. ⚠️ `−90%~+67%`는 CAV 임베딩 개입치라 텍스트 추가/삭제 효과로 **인용 금지** — 방향·상한 증거로만. E-C5 `[PRIMARY-pre2025 · 2024-06 · 배경]` 최상위 모델도 생성 주석의 약 **20%**가 검증가능하게 부정확. **양(positive) 효과 크기 없음(추론)** — O-3(추가의 대칭 효과 미확립).
- **동작 위험**: 없음(주석 미실행). · **사실성 위험**: **~20~45%** — E-C5 약 20% 부정확, E-C4 `[PRIMARY-2025+ · ICSE 2025]` 특화 파인튜닝 모델의 주석 수정도 35~45%가 틀림 → **자동 적용 금지**.
- **1급 센서**: 코드 대조(주석 각 절 ↔ 실제 분기/예외). · **2급 센서**: 가능하면 **document testing** — 주석에서 실행 가능한 검사를 도출해 pass/fail 채점(E-C5, ROC-AUC 0.67·정확도<0.8 필터 시 부정확 주석 46% 제거). 테스트 LLM 자체도 환각 가능하므로 보조 신호로만.
- **적용 도구**: 텍스트 편집. 사실성 위험이 실행으로 완전 해소되지 않으면 **제안만** 하고 사람 확인.
- **롤백**: hunk revert.
- **게이트**: B.
- **거부 조건**: 코드 대조로 검증 불가한 추측성 주장(성능·의도·미래 계획)은 추가하지 않는다. 검증 실패한 계약은 추가 대신 **불확실 표기 후 사람에게 질의**.

## C2 · 안전 리네임

- **id**: `C2` · **이름**: 무의미·오도 식별자 안전 리네임 · **우선순위**: **P2** · **대상 축**: 명명
- **무엇을 바꾸나** — 의미 단서 없는 식별자를 본문이 하는 일과 일치하는 이름으로 바꾼다.
  ```
  d = fetch()          # d = 활성 사용자
  for x in d: notify(x)
  ```
  →
  ```
  active_users = fetch()
  for user in active_users: notify(user)
  ```
- **근거** — E-N1 `[PRIMARY-2025+ · 2025-10]` 명명 채널 억제 시 class-level accuracy **−11~29pt**(통제 perturbation ablation → 인과 정당). E-N3 `[PRIMARY-2025+ · 2025-04, NeurIPS 2025]` 식별자 NL 단서 의존(LiveCodeBench 평균 −8.20pp) — ⚠️ 효과 크기는 **기존 이름이 이미 얼마나 좋은지에 의존**(CRUXEval −2.38pp). E-N4 `[PRIMARY-2025+ · 2025-03]` 좋은 함수명은 구현 복원 수준의 의미를 운반. ⚠️ 이들은 **제거 ablation(감소분)** 이므로 "좋은 이름을 새로 붙이면 같은 크기로 오른다"는 **대칭 가정 불가(O-3)** → 개선 방향 효과 크기 없음(추론).
- **동작 위험**: 참조/컴파일 깨짐 — E-A4 `[PRIMARY-2025+ · 2026-05]` 자동 리팩터의 주 실패는 논리 오류가 아니라 **컴파일 가능성 저하**. · **사실성 위험**: E-N4 false positive — LLM은 **부정확한 이름도 메워서 맞춘다** → **LLM의 코드 이해 성공을 명명 오라클로 쓰지 말 것.** 판정은 이름↔본문 대조(구조적 검사).
- **1급 센서**: **컴파일/타입체크 + 참조 완전성** — 리네임 전후 심볼 참조 수 일치를 LSP로 확인, `tsc --noEmit` / `pyright` / `go build ./...` / `cargo check` 재통과. · **2급 센서**: 텍스트 grep(`git grep -n '\bd\b'`) — ⚠️ 스코프/섀도잉을 구분 못 하므로 advisory only, 절대 grep 치환으로 적용하지 않는다.
- **적용 도구**: **AST/LSP 기반 rename만** — Python: rope·pyright/LSP, TS/JS: ts-morph·tsserver LSP rename, Go: gopls rename, Rust: rust-analyzer, Java: IDE LSP. **도구가 없으면 적용하지 말고 제안만.**
- **롤백**: 동일 도구로 역방향 rename(새→옛). safe by construction(E-A5: 잘못된 연산은 그냥 실패).
- **게이트**: B.
- **거부 조건**: **공개 API·직렬화 키·DB 컬럼·외부 계약(설정 키·이벤트 이름·리플렉션 참조)** 에 노출된 심볼 — 컴파일러가 못 잡는 **런타임 파괴**를 일으킨다. AST/LSP 스코프 밖 문자열·직렬화 경계는 반드시 **사람 판단**(E-A5: developer supervision is not optional).

## C3 · 구조 리팩터

- **id**: `C3` · **이름**: 구조 리팩터(경계·응집·추출) · **우선순위**: **P4 · 기본 OFF · opt-in** · **대상 축**: 구조
- **무엇을 바꾸나** — 뒤섞인 책임을 **최소 단위**로만 분리한다(통째 재작성 금지).
  ```
  def handle(req):
      # 검증 + 파싱 + 저장이 한 덩어리
      ...
  ```
  →
  ```
  def handle(req):
      valid = validate(req)      # 추출된 최소 단위만
      return persist(parse(valid))
  ```
- **근거** — E-S1 `[PRIMARY-2025+]` 강한 정량 델타는 전부 **툴 측 그래프/인덱스** 효과지 **본문 수정 측정이 아니다.** 본문 구조 개선 효과는 **추론이지 측정이 아니다 → 효과 크기 없음(추론)**. ⚠️ ARISE는 `+4.7pp` end-to-end만 인용(R@1 수치는 REFUTED). E-S2 `[PRIMARY-2025+ · 2026-03]` 구조는 최대 실패면 — micro 53.5% → **multi-step 20.6%**, 의존성 통제 4.3%, **64/483(13.3%)이 기능 테스트 통과하고도 구조 오라클 실패**.
- **동작 위험**: **높음** — E-A1 `[PRIMARY-2025+ · 2026-02]` 안전 재작성을 명시해도 **19~35%가 기능적 비등가**. E-A3 단순화조차 ≈26% 비등가(복잡도에 비례, 큰 함수 통째가 가장 위험). · **사실성 위험**: 낮음(주석/이름이 아닌 제어흐름 변경).
- **1급 센서**: 테스트 + 컴파일 **+ 사람 diff 리뷰(필수)** — ⚠️ **"테스트 통과 = 동작 보존"이 아니다**: E-A2 테스트가 비등가의 **≈21%를 놓친다**(HumanEval 20.72·MBPP 21.65·APPS 22.41%). 등가 증명 불가를 명시하고 사람이 최종 결정. `pytest` / `go test ./...` / `cargo test` + 리뷰어 diff 정독. · **2급 센서**: 가능 시 differential fuzzing(E-A2 권고), reward-hacking 감시(E-M1·E-M2, trace만으로는 28% 놓침).
- **적용 도구**: LSP의 extract/move 리팩터 API가 있으면 그것으로만, 없으면 **제안만**. 에이전트 자율 적용 금지(E-S2: 의존성 통제 통과율 4.3%).
- **롤백**: 전체 diff `git revert`.
- **게이트**: B **+ C3 전용 사람 diff 리뷰 필수**.
- **거부 조건**: (1) **테스트 스위트 부재** 시 제안 자체 금지, (2) **큰 함수 통째 재작성**, (3) **다단계 연쇄 변경**(E-S2 multi-step 20.6% — 한 번에 한 최소 단위만).

## 제안 랭킹 규칙

Phase 2 제안표는 각 개입 후보를 **위험조정 기대이익**으로 정렬한다:

```
위험조정 기대이익 = (근거 강도 × 효과 방향) ÷ (동작 위험 + 사실성 위험)
```

- **근거 강도**: SOURCE-TIER + 효과 크기(측정 > 추론). 측정된 감소분(제거 ablation)은 방향 근거로 쓰되 개선 방향은 **추론**으로 감쇄한다(O-3).
- **효과 방향**: 오도 제거(확실한 +) > 검증된 명료화 > 미검증 추가.
- **분모**: 동작 위험 + 사실성 위험. C0은 분모≈0이라 효과 크기가 작아도 최상위(P1이 1순위인 이유 = 위험 0·확실). C3는 분모가 최대라 효과가 추론이면 최하위(P4가 마지막인 이유).
- **동률 처리**: 기대이익이 같으면 **P 번호가 낮은 쪽 우선**(P1 > P2 > P3 > P4).
- **스코프 가드**: 사용자가 Phase 0에서 선택한 개입 클래스 **밖**의 후보는 제안표에 점수를 매기지 않고 **"추가 안 함(스코프 밖)"** 으로 표기하며, **확장 여부를 사용자에게 문의**한다(임의 적용 금지). 이 플러그인은 등급이 아니라 **census + 개입 제안 + 승인 후 적용**만 낸다.
