# 플래너 스킬 실전 예제

## 예제 1: 기능 구현 PRD

```
/plan JWT 기반 사용자 인증 구현
```

**인터뷰 단계:**
- Q: "이 기능이 해결하는 문제는 무엇인가요?"
- A: "사용자가 안전하게 로그인해야 합니다"
- Q: "범위는 어디까지인가요?"
- A: "기본 로그인/로그아웃 + 토큰 리프레시"
- Q: "제약 조건이 있나요?"
- A: "기존 User 모델 사용, OAuth 제외"

**조사 단계:**
- 관련 파일: `src/models/User.ts`, `src/routes/index.ts`
- 현재 패턴: Express 사용, 인증 미들웨어 없음

**계획 생성:**
- 6개 구현 단계
- 4개 수정 대상 파일
- 4개 인수 조건

**승인:**
사용자에게 계획을 제시하고 승인 요청.
사용자: "승인"

**출력:**
```
PRD가 .claude/plans/user-authentication-jwt.md에 저장되었습니다.
```

## 예제 2: 리팩토링 PRD (수정 요청 포함)

```
/plan API에 의존성 주입 패턴 적용
```

**인터뷰 단계:**
- Q: "어떤 서비스에 DI를 적용할 건가요?"
- A: "src/services/ 하위 모든 서비스"
- Q: "테스트에 미치는 영향은?"
- A: "모킹이 더 쉬워져야 합니다"

**조사 단계:**
- 관련 파일: 8개 서비스 파일, 현재 직접 인스턴스 생성 패턴
- 현재 패턴: DI 컨테이너 없음, 직접 인스턴스화

**계획 생성:**
- 12개 구현 단계
- 컨테이너 설정, 서비스별 리팩토링

**승인:** 사용자가 "수정 요청" 선택 → "5단계를 팩토리 패턴으로 변경해주세요"

**수정된 계획:** 5단계 수정 후 재제시

**2차 승인:** 사용자 "승인"

**출력:**
```
PRD가 .claude/plans/api-dependency-injection.md에 저장되었습니다.
```

## 예제 3: 테스트 커버리지 PRD

```
src/services/ 테스트 커버리지 추가 PRD 작성해줘
```

**인터뷰 단계:**
- Q: "단위 테스트만, 아니면 통합 테스트도?"
- A: "둘 다"
- Q: "커버리지 목표는?"
- A: "최소 80%"

**조사 단계:**
- 관련 파일: 6개 서비스 파일, 기존 테스트 패턴 확인
- 현재 패턴: Jest 설정 존재, 2개 서비스에만 테스트 있음

**계획 생성:**
- 테스트 미작성 서비스별 테스트 파일
- 통합 테스트 스위트
- 커버리지 설정

**승인:** 사용자 "승인"

**출력:**
```
PRD가 .claude/plans/service-test-coverage.md에 저장되었습니다.
```
