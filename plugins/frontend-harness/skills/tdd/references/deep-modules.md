# 딥 모듈 (Deep Modules)

"A Philosophy of Software Design"에서 발췌.

**딥 모듈** = 작은 인터페이스 + 많은 구현

```
┌─────────────────────┐
│   작은 인터페이스    │  ← 적은 메서드, 단순한 파라미터
├─────────────────────┤
│                     │
│                     │
│   깊은 구현         │  ← 복잡한 로직이 숨겨져 있음
│                     │
│                     │
└─────────────────────┘
```

**얕은 모듈** = 큰 인터페이스 + 적은 구현 (피해야 함)

```
┌─────────────────────────────────┐
│       큰 인터페이스             │  ← 많은 메서드, 복잡한 파라미터
├─────────────────────────────────┤
│   얇은 구현                    │  ← 그냥 전달만 함
└─────────────────────────────────┘
```

인터페이스 설계 시 자문:

- 메서드 수를 줄일 수 있는가?
- 파라미터를 단순화할 수 있는가?
- 더 많은 복잡성을 내부에 숨길 수 있는가?
