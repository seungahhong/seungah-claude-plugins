# Research — 요구사항 분석 리서처 커맨드

요구사항을 체계적으로 분석하고, 누락된 정보를 발굴하여 명확한 요구사항 명세를 도출하는 커맨드.

## 역할

사용자가 제시한 요구사항이나 아이디어를 받아, grill-me 스킬을 서브에이전트로 실행하여 모호한 부분을 파고들고, 코드베이스와 외부 자료를 조사하여 실행 가능한 수준의 요구사항 명세로 정제한다.

## 작업 흐름

### 1단계: 초기 분석

입력된 요구사항을 읽고 다음을 파악한다:

- **명시된 요구사항** — 사용자가 직접 언급한 것
- **암묵적 요구사항** — 문맥상 당연히 필요하지만 언급되지 않은 것
- **모호한 영역** — 여러 해석이 가능하거나 정보가 부족한 부분 (**내부적으로만 파악, 사용자에게 보여주지 않는다**)
- **기술적 제약** — 코드베이스, 기존 아키텍처, 의존성에서 오는 제약

**사용자에게 보여주는 항목:** 명시된 요구사항, 암묵적 요구사항, 기술적 제약만 보여주고 수정·보완 의견을 확인한다.

> **금지:** 모호한 영역 목록을 사용자에게 절대 보여주지 않는다. "모호한 영역 N가지"라고 언급하는 것도 금지한다. 모호한 영역은 2단계 인터뷰에서 **한 번에 1개씩** 질문으로 해소한다.

### 2단계: grill-me 서브에이전트 실행 — 인터뷰

Agent 도구를 사용하여 **grill-me 서브에이전트**를 spawn한다.

**서브에이전트에게 전달할 프롬프트 구성:**
```
1. 사용자의 요구사항 원문
2. 1단계에서 파악한 초기 분석 결과 (명시된/암묵적/모호한 영역/기술적 제약)
3. 스킬 경로: ./skills/grill-me/SKILL.md — 이 스킬의 프로세스를 따를 것
4. 코드베이스 컨텍스트 (현재 프로젝트 경로, 관련 파일 정보)
5. 산출물 요청: 모호한 영역에 대한 질문 목록과 사용자 답변을 수집하여, 확정된 요구사항 목록을 반환할 것
```

**서브에이전트 핵심 지시:**
- **반드시 한 번에 1개만 질문한다** — 절대로 2개 이상의 질문을 동시에 던지지 않는다. 모호한 영역이 7개여도 1개씩 순차적으로 묻는다. 질문 목록 전체를 미리 보여주는 것도 금지한다.
- 코드베이스에서 답을 찾을 수 있는 것은 직접 조사한 뒤 확인만 요청한다
- 이전 답변과 모순되는 부분이 발견되면 즉시 지적한다
- 답변에 따라 다음 질문을 동적으로 결정한다 (미리 정한 목록을 순서대로 묻지 않는다)

**서브에이전트가 사용자에게 질문해야 하는 경우:**
서브에이전트의 질문을 그대로 사용자에게 전달하고, 사용자의 응답을 서브에이전트에 전달한다. 이 과정을 모든 핵심 질문이 완료될 때까지 반복한다.

**수집할 산출물:**
- 확정된 요구사항 목록 (사용자 답변이 반영된 것)
- 결정 로그 (어떤 질문에 어떤 답변이 왔는지)

### 3단계: 리서치

인터뷰 결과를 바탕으로 다음을 조사한다:

- **코드베이스 탐색** — 관련 파일, 기존 구현, 패턴, 의존성 파악
- **기술 조사** — 필요한 경우 외부 문서, API 스펙, 라이브러리 호환성 확인
- **유사 사례 분석** — 코드베이스 내 비슷한 기능이 어떻게 구현되었는지 참고

### 4단계: 요구사항 명세 작성

모든 질문과 조사가 완료되면, 아래 형식으로 요구사항 명세를 정리한다.

```markdown
## 요구사항 분석 결과

### 개요
[프로젝트/기능에 대한 1-2문장 요약]

### 기능 요구사항
| ID | 요구사항 | 우선순위 | 비고 |
|----|----------|----------|------|
| FR-01 | ... | 높음/중간/낮음 | ... |

### 비기능 요구사항
| ID | 요구사항 | 기준 |
|----|----------|------|
| NFR-01 | ... | ... |

### 기술적 제약 및 의존성
- ...

### 미해결 사항
- ...

### 결정 로그
| 항목 | 결정 | 근거 |
|------|------|------|
| ... | ... | ... |
```

### 5단계: 확인

작성된 명세를 사용자에게 제시하고 확인을 요청한다. 수정이 필요하면 2단계로 돌아가 grill-me 서브에이전트를 다시 spawn하여 추가 질문을 진행한다.

## 사용 시 유의사항

- 이 커맨드는 **분석과 질문**에 집중한다. 설계나 구현은 다른 커맨드(prd, frontend-guidelines 등)의 역할이다.
- 사용자가 바쁘거나 간결한 답변을 선호하면, 질문 수를 줄이고 합리적인 기본값을 제안한 뒤 확인만 받는다.
- 코드베이스 조사 결과를 질문에 녹여, 사용자가 맨땅에서 답하지 않아도 되게 한다.
