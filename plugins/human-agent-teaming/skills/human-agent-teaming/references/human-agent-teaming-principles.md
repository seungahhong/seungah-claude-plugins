# Human-Agent Teaming — 원리 · 분업 규칙 · 감독 설계 · anti-pattern

> 이 문서는 human-agent-teaming 하네스가 따르는 설계 원리의 근거다. 오케스트레이터(SKILL.md)와 각 에이전트가 참조한다.
> 출처·인용·신뢰도·CAVEAT·반박된 주장은 [human-agent-teaming-research.md](./human-agent-teaming-research.md)에 정리돼 있고,
> 이 문서의 원칙과 상호 참조된다(§N은 research dossier의 절 번호). 빠르게 변하는 분야이므로 수치는 vote/CAVEAT와 함께만 인용한다.

## 1. 무엇이 human-agent teaming 설계인가

이 하네스의 첫 질문은 "에이전트에게 무엇을 시킬까(프롬프트)"가 아니라 **"사람과 에이전트가 *한 팀으로* 어떻게 일할까"** 이다.
일은 단일 플레이어에서 **멀티플레이어**로 옮겨갔다 — *사람이 전략을 세우고 에이전트가 실행*한다(§1). 그래서 산출물은
프롬프트가 아니라 **협업 설계(teaming playbook)** 다 — 분업·공통기반·감독/신뢰·검증/지속의 *working agreement* 집합.

**제1원칙(중심 긴장): 자율성 ⇄ 통제.** 에이전트는 자율적으로 일해야 하고(그 독립성이 가치다), 그러나 인간은 *목표가
어떻게 추구되는지*에 대한 통제를 — 특히 **고위험·비가역 결정 이전에** — 유지한다(§2). 이 하네스의 네 단계는 이 긴장을
작업 종류별로 *해소*하는 절차다.

> 이 하네스는 **협업을 *설계*한다(제안·playbook 산출)** — 실제 협업을 대신 실행하지 않는다. 모든 단계는 사람 승인 게이트로 시작한다.

## 2. 분업 규칙 (division of labor)

### 2.1 멀티플레이어 분업 — 사람 vs 에이전트
효과적 분업은 역할을 *명문화*한다(§1). 기본 배치:

- **사람이 소유**: 전략·북극성(north star), *역할 정의(who does what)*, **하드 트레이드오프 의사결정**, **최종 검증**,
  그리고 *책임(accountability)*.
- **에이전트가 소유**: 구별된 *전문 실행 역할*(예: 데이터 분석 / 표준 집행 / 리서치 종합) — roster + skill 파일로 명문화.
- 명문화 원칙: **"적혀 있고 접근 가능하지 않으면, 에이전트에겐 존재하지 않는다"**(공개적으로 일하라, §1).

### 2.2 위임 메커니즘 — 무엇을, 누구에게, 얼마나
위임은 세 메커니즘 + 실패 대비로 구조화한다(§8): **human delegation · agent delegation · co-delegation**, 그리고 위임
실패 시 **deferred/backup**. 효과적 agent delegation은 **AI 오류 경계(parsimony·stochasticity)를 드러내** 사람이 정확한
mental model을 갖게 하고, **conditional delegation** — 사람·에이전트가 *trustworthy region*을 협상하고 그 너머를 위임 —
으로 *역량 경계*를 못 박는다. 운영 모드는 작업 중요도×신뢰성에 따라 **human-in-the-loop**(사람이 모든 최종 결정) vs
**human-on-the-loop**(에이전트 자율 + 사람이 결과 감독)을 고른다(§12).

### 2.3 명확한 역할 경계 — restrained 에이전트
**흐릿한 역할 경계는 실패 원천**이다 — 역할 혼동, 의사결정 지연·오류, 신뢰 하락, **인간 무임승차**, **과신**(§9). 따라서
*명확한 역할 정의를 가진 제한된(restrained) 에이전트*로 **인간 주도성을 보존**한다. 에이전트가 사람 역할(특히 하드
트레이드오프·최종 책임)을 *대신 가져가지 않게* 경계를 고정한다.

## 3. 공통기반 (common ground) — 일회성이 아니라 영속

에이전트는 *체험 경험이 없다* — 공통기반은 **만들고 지속 재협상**해야 한다(§11). 좋은 협업의 시작은 *팀원 온보딩*에 가깝다:

- **브리핑(온보딩)**: 북극성·제약·"좋은 결과의 모습"·자료 위치·예시·질문/에스컬레이션 방법을 *적어서* 제공(§1).
- **AI 오류 경계 노출**: 에이전트가 어디서 틀리는지(parsimony·stochasticity)를 드러내 사람의 mental model을 보정(§8).
- **workspace awareness**: 에이전트의 *초점·의도*가 보이게 한다(실시간 계획/to-do) — 감독의 전제(§4·§11).
- **지속 재정렬**: SMM은 미리 정의되지 않고 *영속 협상*된다(§10). 일회성 컨텍스트 설정이 아니라 **재정렬 루프**(주기적
  plan-restatement · divergence 감지 · After-Action Review/Explanation)를 둔다.

> ⚠️ 경계: 이것은 *모델 컨텍스트 윈도우에 넣을 토큰 페이로드의 조립·압축*(retrieval·큐레이션)이 **아니다**. 여기서
> 다루는 것은 *협업 수준의 공통기반·working agreement*다(팀원이 알아야 할 것·합의·재정렬). 토큰 단위 최적화는 컨텍스트
> 설계 도메인의 일이다.

## 4. 감독 = 모니터링 + 개입 (approve-everything가 아니다)

효과적 감독은 **모든 행동 승인을 요구하지 않는다** — *필요할 때 개입할 위치*에 있는 것이다(§4). 모든 행동 승인 강제는
*승인 피로·러버스탬프*로 **마찰만 늘고 안전 이득은 없을 수 있다**. 모니터링 기반 감독의 1급 요소:

1. **투명성(transparency)** — 실시간 to-do/계획으로 에이전트 행동을 노출. *너무 적으면 감독 불능, 너무 많으면 압도* — 그
   사이를 잡는다.
2. **개입 가능성/스티어링(interruptibility)** — 언제든 중단·재지정·작업계획 조정.
3. **단계적·가역 권한(graduated, reversible)** — 기본 읽기 전용(무승인) → 쓰기/수정 전 사람 승인 → 신뢰된 루틴엔 선택적
   영속 권한. *비가역 행동 이전엔 항상 체크포인트*.
4. **위험·신뢰 기반 트리거** — confidence floor·action-risk tier 등으로 *언제 사람을 부를지* 명문화(에스컬레이션).

### 4.1 신뢰 보정 = 적절한 의존 (신뢰 최대화가 아니다)
목표는 **appropriate reliance** — *과신은 위험, 과소신은 무시*(§3). 신뢰는 *작업 종류별로 점진* 획득한다 — 처음엔 모든
산출물 검토 → 반복 성공 후 의도적 확대. 검증 강도는 **stakes × uncertainty에 비례**한다(저위험·고신뢰는 가볍게,
고위험·저신뢰는 무겁게).

### 4.2 감독이 *무너지는* 경우를 가드한다
모니터링 기반 감독은 *사람이 실제로 본다*를 전제한다. 그 전제가 깨지는 **러버스탬프·승인 피로·자동화 편향**을 명시적으로
가드한다 — 주기적 강제 리뷰, 비가역/고위험엔 승인 *간소화 금지*, interrupt-rate 같은 *능동 모니터링 신호* 권장(§4
openQuestion). ⚠️ 이는 *완화책*이지 만능 해결이 아니다(약속 금지).

## 5. 검증 = 요구되는 마지막 층 (operationalized)

**인간 검증은 자동 검증이 가능할 때조차 마지막 validation 층**이다(§5). 막연한 "리뷰"가 아니라 *구체적 산출물*로 운영한다:

- **코드** → 테스트(기능 검증). **그 외 작업** → **루브릭/스타일 가이드**.
- **Doer-Verifier 패턴** → 작업 에이전트와 *별도의* 검사 에이전트(이상적으로 **fresh-context + Write/Edit 없는** 평가자).
- **검증 스캐폴딩** → 에이전트의 *조기 완료 선언*을 막는다: 브라우저 자동화 제공, "사람 사용자처럼 테스트", "신중히
  테스트한 뒤에만 passing 표시"(§6).
- **대칭 전문성 문제** → 검증 비용=생산 비용이면(전문가만 검증 가능) *검증 가능한 하위 주장으로 분해·ground-truth
  프로브·독립 컨텍스트 Doer-Verifier*로 비용을 낮춘다(§5 CAVEAT).

> ⚠️ **검증 스캐폴딩은 필요조건이지 충분조건이 아니다.** 에이전트 종단 성공률<100%·도구 사각(예: 일부 자동화는 네이티브
> 모달을 못 봄)이 있어, 검증을 두어도 *통과를 단정하지 않는다*. "AI가 AI를 검증하는 폐루프"의 한계 때문에 *인간 층*이 남는다(§5).

## 6. 지속 (sustain) — 핸드오프 연속성 + 후속 재정렬 + 책임

- **핸드오프 연속성 산출물**(§7): fresh-context 작업자가 상태를 빨리 이해하도록 *진행 로그 · 구조화된 기능/요구
  목록(passes:true/false) · 서술적 커밋 이력*. 비유: *기억 없는 교대 근무자*.
- **후속 재정렬(SMM)**: After-Action Review/Explanation 등으로 *무엇이·왜 그렇게 됐는지* 사후 공유 → 다음 회차 mental
  model 개선(§10). 자율은 *작업 종류별로* 반복 성공 후 확대(§3).
- **책임(accountability)**: 에이전트는 *팀원이지 희생양("moral crumple zone")이 아니다* — 결과 책임은 사람이 진다. 분업이
  책임 회피의 구실이 되지 않게 한다(§9 무임승차 가드와 연결).

## 7. anti-pattern (피해야 할 것)

- **single-player 사고** — 팀(분업·역할)을 설계하지 않고 한 에이전트에 *프롬프트만* 던진다(§1).
- **more-trust/more-autonomy-is-better** — 신뢰·자율은 많을수록 좋다는 가정. 목표는 *적절한 의존*이다(과신은 위험, §3·§9).
- **approve-everything를 안전으로 착각** — 행동별 전수 승인 강제 → 승인 피로·러버스탬프, 마찰만 늘고 안전 이득 없음(§4).
- **에이전트 자기제한에 안전을 의존** — 에이전트가 알아서 멈춰줄 것이라 믿는다(이 주장은 **반박됨**, dossier 반박 §). 감독은
  *사람 모니터링+개입·구조적 가드*에 둔다.
- **흐릿한 역할 경계** — 누가 무엇을·누가 최종 책임을 안 정해 역할 혼동·무임승차·과신을 부른다(§9).
- **일회성 컨텍스트 설정** — 공통기반을 처음 한 번만 주고 *재정렬 루프*가 없다(SMM은 영속 협상, §10·§11).
- **무경계 위임(범위 폭주)** — 한 번에 너무 많이 시켜 *조기 승리 선언*을 부른다 → *한 번에 하나의 경계 작업*(§6).
- **검증 없는 완료 수용** — "다 됐다"는 자기보고를 검증 산출물 없이 믿는다(§5·§6). 스캐폴딩 없이는 미검증 완료가 새어 든다.
- **검증을 충분조건으로 착각** — 테스트/검증을 두면 *반드시* 옳다고 단정한다(성공률<100%·도구 사각, §5).
- **벤더 수치 과신** — "dramatically improved" 류 크기 주장을 증명된 효과로 인용한다(§머리말 정직성 가드).
- **책임 전가(moral crumple zone)** — 실패 책임을 에이전트로 떠넘긴다(책임은 사람, §9·§11).

## 8. 결정 신호 요약 (한눈에)

| 신호 | 더 *사람 쪽*(낮은 자율/무거운 감독) | 더 *에이전트 쪽*(높은 자율/가벼운 감독) |
|------|-----------------------------------|----------------------------------------|
| 작업 중요도(stakes) | 고위험·비가역 → 사람 결정·체크포인트 | 저위험·가역 → 위임 가능 |
| 입증된 신뢰성(이 작업 종류) | 낮음/미입증 → 모든 산출물 검토 | 반복 성공 → 범위 확대 |
| 검증 비용 | 검증=생산 비용(대칭 전문성) → 분해·프로브 | 싸게 검증 가능(테스트·루브릭) → 위임 |
| 가역성 | 비가역 → 사전 승인 필수 | 가역(읽기·되돌릴 수 있음) → 무승인/사후 |
| 역할 경계 | 하드 트레이드오프·최종 책임 → 사람 | 전문 실행 → 에이전트(restrained) |
| 감독 모드 | human-in-the-loop(모든 최종 결정) | human-on-the-loop(결과 감독) |

> 모든 신호는 *경향*이지 결정론이 아니다(dossier CAVEAT). 신호가 충돌하면 **고위험·비가역 쪽으로 보수적**으로 기울고,
> 자율은 *반복 성공 후 작업 종류별로* 확대한다. 벤더 1차 근거는 *설계 의도*로, 학술 근거는 *권장 메커니즘*으로 쓴다.
