# 리뷰 프로세스 상세

Phase 2(Commit 리뷰)와 Phase 3(PR 리뷰)에서 공통으로 사용하는 리뷰 실행·수정 절차.

---

## 리뷰 대상 diff

| 단계 | diff 명령 | 설명 |
| ---- | --------- | ---- |
| Phase 2 (Commit) | `git diff HEAD` | 현재 변경사항 (staged + unstaged) |
| Phase 3 (PR) | `git diff origin/<target-branch>..HEAD` | 타겟 브랜치 대비 전체 변경 |

---

## 병렬 실행

`/simplify`와 `/review`를 **하나의 어시스턴트 메시지 안에서 동시에 Agent spawn**한다. 순차 실행 금지.

### /simplify 서브에이전트

Agent 도구로 `code-simplifier` 서브에이전트를 spawn한다.

- 대상 파일의 명확성·중복·가독성을 점검하고 **개선을 직접 적용**한다
- 기능 변경 금지 — 동작 동일성 유지
- 산출물: 적용된 변경 diff, 적용하지 않은 제안 목록

### /review 서브에이전트

Agent 도구로 리뷰 서브에이전트를 spawn한다. 프롬프트 안에서 빌트인 `/review` 스킬을 호출하도록 지시한다.

- 코드 품질, 컨벤션, 로직 버그, 누락된 케이스 점검
- 대상 파일 전체를 리뷰한다
- 산출물: Critical / Important / Suggestion 분류된 이슈 목록

---

## 결과 분류 및 수정

리뷰 결과에서 발견된 이슈를 **자동 수정**과 **사용자 확인 필요**로 분류한다.

### 자동 수정 (사용자 확인 없이 적용)

다음에 해당하는 이슈는 즉시 수정을 적용한다:

- 코드 스타일: 포맷팅, 네이밍 컨벤션, 불필요한 공백/줄바꿈
- /simplify 간소화: 중복 코드 제거, 가독성 개선, 불필요한 복잡도 제거
- 단순 코드 수정: 미사용 import/변수 제거, 명백한 타입 오류

자동 수정 후 적용된 변경 내역을 간략히 보고한다:

```
[Review] 자동 수정 N건 적용 완료

| 파일 | 변경 요약 |
| ---- | -------- |
| src/components/Button.tsx | 미사용 import 제거, 포맷팅 정리 |
| src/hooks/useAuth.ts | 중복 로직 간소화 |
```

### 사용자 확인 필요 (출력 후 선택)

다음에 해당하는 이슈는 사용자에게 보여주고 수정 여부를 묻는다:

- 로직 변경이 수반되는 수정 (조건문 변경, 분기 추가/제거)
- 인터페이스/API 시그니처 변경
- 동작이 달라질 수 있는 리팩토링
- 의존성 변경
- Critical/Important 등급의 버그 수정

```
[Review] 사용자 확인이 필요한 이슈가 N건 있습니다.

| # | 심각도     | 파일:라인 | 이슈 | 수정 제안 |
|---|-----------|----------|------|----------|
| 1 | Critical  | ...      | ...  | ...      |
| 2 | Important | ...      | ...  | ...      |
| 3 | Suggestion| ...      | ...  | ...      |

각 항목에 대해 선택해주세요:
- 수정: 항목 번호 (예: "1, 2")
- 전체 수정: "전체"
- 건너뛰기: "skip" — 수정 없이 다음 단계로 진행
```

선택 해석:

| 입력 | 해석 |
| ---- | ---- |
| 번호 목록 (`1, 3`) | 해당 항목만 수정 |
| "전체" / "all" | 모든 항목 수정 |
| "skip" / "건너뛰기" / "패스" | 수정 없이 다음 단계 |
| 빈 응답 / 범위 밖 번호 | 다시 안내 |

수정 적용 후 변경 diff를 사용자에게 보여준다.

---

## 안티패턴

- ❌ /simplify 먼저 끝내고 /review를 시작하는 순차 실행
- ❌ 자동 수정 대상을 사용자에게 일일이 확인 — 간단한 스타일/코드 수정은 바로 적용
- ❌ 사용자 확인 필요 항목을 무단으로 자동 수정 — 로직 변경은 반드시 확인
- ❌ /simplify 결과를 /review의 입력으로 넣기 — 독립적으로 평가
