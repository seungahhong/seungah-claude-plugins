# git-harness

Git 워크플로우를 지원하는 Claude Code 멀티 에이전트 플러그인입니다. 한국어 커밋 메시지 작성과, 리뷰를 내장한 커밋·PR 생성 올인원 워크플로우를 제공합니다.

## 설치

```bash
claude plugin add seungahhong/seungah-claude-plugins
```

## 스킬

| Skill | Command | 설명 |
|-------|---------|------|
| Commit | `/commit` | `이슈번호 type: 제목` 형식의 한국어 커밋 메시지 작성. 명령형 제목 + 필요 시 요약/영향/테스트 시나리오 본문 |
| Review to PR | `/review-to-pr` | 변경 파일 기반 **리뷰 → 커밋 → PR 생성** 올인원. 각 단계에서 `/simplify` + `/review`를 자동 수행하고 수정 적용 |

## 사용법

### 1. 커밋 메시지만 작성 — `/commit`

스테이징/변경된 diff를 읽어 한국어 커밋 메시지를 작성하고 커밋합니다.

```
/commit
```

- 형식: `이슈번호 type: 제목` (예: `#123 feat: 로그인 폼 검증 추가`)
- 제목은 한국어 명령형으로 작성하고, 변경이 크면 요약·영향·테스트 시나리오 본문을 덧붙입니다.
- Secrets 검사와 보호 브랜치 경고를 포함합니다.
- 자연어로도 호출됩니다: "커밋해줘", "변경사항 커밋", "commit".

### 2. 리뷰부터 PR까지 한 번에 — `/review-to-pr`

변경 파일을 대상으로 리뷰를 내장한 커밋·PR 생성을 한 흐름으로 처리합니다.

```
/review-to-pr
```

흐름:

```
변경 파일 수집
  → [Commit 단계]  /simplify + /review 자동 리뷰 → 수정 적용 → 커밋
  → [PR 단계]      /simplify + /review 자동 리뷰 → 수정 적용 → PR 생성
```

- 실행할 단계(Commit / PR)를 선택할 수 있습니다.
- 각 단계 안에서 코드 간소화·PR 리뷰가 자동으로 돌고, 발견된 수정사항을 반영한 뒤 커밋 또는 PR을 만듭니다.
- 자연어로도 호출됩니다: "PR 준비", "PR 올려줘", "리뷰하고 커밋하고 PR", "머지 준비".

> 두 스킬은 모두 모델 자동 호출이 비활성화돼 있어, 슬래시 커맨드 또는 위 키워드로 **명시 호출**해야 동작합니다.
