# 플랫폼별 랭킹 요소 가이드

각 검색 플랫폼이 콘텐츠를 평가하는 방식과 최적화 전략.

## AI 대화형 검색 (LLM 기반 검색)

대화형으로 질문에 답변하며 웹 콘텐츠를 인용하는 서비스.

**랭킹 요소 비중:**
- 권위성 & 신뢰성: 40%
- 콘텐츠 품질: 35%
- 플랫폼 신뢰도: 25%

**핵심 통계:**
- 자사 도메인이 서드파티보다 11% 더 많이 인용됨
- 30일 이내 업데이트 콘텐츠 = 인용률 3.2배 상승
- 고품질 참조 도메인 350K+ = 평균 8.4회 인용

**최적화 방법:**
- [ ] 브랜드 도메인 권위 구축 (자사 도메인에 콘텐츠 집중)
- [ ] 콘텐츠 신선도 유지 (최소 월 1회 업데이트)
- [ ] 백링크 확보 전략 실행
- [ ] AI 응답 형식에 맞는 콘텐츠 구조화

## AI 검색 엔진 (RAG 기반)

검색 결과를 기반으로 AI가 답변을 생성하는 전용 검색엔진.

**아키텍처:** 3단계 재랭킹 시스템
- L1: 기본 관련성 매칭
- L2: 전통적 SEO 요소 평가
- L3: ML 기반 품질 평가

**핵심 요소:**
- FAQ 스키마 → 가시성 상승
- 의미적 관련성 > 키워드 매칭
- PDF 문서 우대 (인용 우선순위)
- 콘텐츠 신선도

**최적화 방법:**
- [ ] robots.txt에서 전용 크롤러 봇 허용
- [ ] FAQ 섹션 + FAQPage 스키마 추가
- [ ] 핵심 문서를 PDF로도 제공
- [ ] 키워드보다 주제의 의미적 깊이에 집중

## 전통 검색엔진 AI 답변 (SGE/AI Overview)

전통 검색엔진이 검색 결과 상단에 AI 생성 답변을 표시하는 기능.

**소스 선정 파이프라인:** 5단계
1. 후보 페이지 수집
2. 관련성 필터링
3. E-E-A-T 평가
4. 콘텐츠 추출
5. 최종 소스 선정

**핵심 통계:**
- 검색의 85%+ 에 AI 답변이 표시됨
- 전통적 상위 10위와 AI 답변 소스의 겹침은 15%에 불과

**최적화 방법:**
- [ ] E-E-A-T 최적화 (경험, 전문성, 권위성, 신뢰성)
- [ ] 구조화 데이터(JSON-LD) 마크업 필수
- [ ] 주제 권위 구축 (콘텐츠 클러스터 + 내부 링크)
- [ ] 권위 있는 인용 추가 (+132% 가시성)

## 생태계 연동 검색

특정 기술/비즈니스 생태계 내에서 작동하는 AI 검색.

**핵심 요소:**
- 해당 생태계의 검색 색인에 등록 필수
- 생태계 내 플랫폼(관련 커뮤니티, 코드 저장소 등) 언급이 랭킹에 영향
- 페이지 로딩 2초 이내 필수
- 명확한 엔티티 정의

**최적화 방법:**
- [ ] 관련 생태계 검색엔진에 사이트 제출
- [ ] 생태계 내 프로필/저장소 관리
- [ ] 페이지 속도 최적화
- [ ] 콘텐츠에 명확한 엔티티 구조 적용

## 독립 AI 어시스턴트

주류 검색엔진이 아닌 대안 검색 인프라를 사용하는 AI 서비스.

**핵심 특성:**
- 주류 검색엔진 대신 대안 검색엔진 색인 사용
- 극도로 선별적 (크롤 대비 인용 비율이 매우 낮음)
- 사실 밀도가 높은 콘텐츠 선호
- 구조적 명료성 중시 (추출하기 쉬운 형태)

**최적화 방법:**
- [ ] 대안 검색엔진에 사이트 제출/확인
- [ ] 데이터와 사실로 채워진 고밀도 콘텐츠 작성
- [ ] 명확한 구조 (H2/H3, 목록, 표)로 추출 용이하게
- [ ] robots.txt에서 해당 봇 허용

## 전통적 검색엔진 SEO

변하지 않는 핵심 랭킹 요소:

| 요소 | 중요도 |
|------|--------|
| 백링크 프로필 | 매우 높음 |
| E-E-A-T 시그널 | 매우 높음 |
| 콘텐츠 품질 & 관련성 | 매우 높음 |
| 페이지 경험 (Core Web Vitals) | 높음 |
| 모바일 우선 | 높음 |
| 검색 의도 매칭 | 높음 |

**Core Web Vitals 기준:**
- LCP (최대 콘텐츠 페인트): < 2.5초
- FID (첫 입력 지연): < 100ms
- CLS (누적 레이아웃 이동): < 0.1
