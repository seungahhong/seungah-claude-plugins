## 근거 서열 (판정의 뼈대)

> **명명·주석 축 ≫ 구조 축.**
> 명명·주석 개입은 **코드 본문을 직접 건드리는 개입**이며, 근거들이 동작을 보존한 채 명명/주석 채널만 억제·주입하는 **통제된 perturbation ablation**이라 **인과 주장이 정당**하다(E-N1·E-N2·E-C1).
> 반면 구조 축의 강한 정량 델타(+12% Pass@10, +32.8%, 43.00%, +4.7pp…)는 **전부 기존 코드를 파싱해 만든 "툴 측 그래프/인덱스"의 효과**이지 **코드 본문 수정의 효과가 아니다.** 본문 구조 개선의 효과는 **추론(inference)이지 측정(measurement)이 아니다**(E-S1). 문서 어디서도 수치를 붙이지 않는다.

## 미해결 질문 O-1~O-5 (플러그인이 답한다고 주장하면 안 되는 것)

- **O-1**: 코드 본문 구조(함수 granularity·모듈 경계·디렉토리 배치)를 실제로 리팩터하면 — 외부 그래프 인덱스를 만드는 것 대비 — 에이전트 국소화·수정 성공률이 정량적으로 얼마나 오르는가? 소스-측 ablation으로 격리한 1차 자료 부재.
- **O-2**: 정적 타입 애노테이션의 **독립** 효과 크기? 단독 변수로 분리한 근거 미확보.
- **O-3**: '좋은 명명·주석을 **추가**'하는 것의 **양(positive) 효과 크기**? 현 근거는 대부분 제거 ablation(감소분)이며 **대칭 가정 불가**.
- **O-4**: 주석 밀도의 최적점 — 도움이 되다가 중복·오도로 해로워지는 교차점은? 모델·태스크별로 어떻게 다른가?
- **O-5**: 명명·주석·구조 개입의 상호작용(가법적인가 포화하는가), 그리고 델타 재측정 시 명명이 정답 단서를 노출하는 reward-hacking을 어떻게 차단하는가?

## 넣지 말 것 (정직성 가드 — 위반 시 근거 충돌)

1. 🚫 "구조를 리팩터하면 에이전트 성공률이 N% 오른다" — **직접 측정 1차 자료 없음**(E-S1). '추론'으로만 표기.
2. 🚫 arXiv:2505.10443의 **"최대 70%"를 리네이밍 효과로 인용** — 루프 언롤링 값이다(E-N5).
3. 🚫 arXiv:2512.16790의 **"−90%~+67%"를 주석 텍스트 추가/삭제 효과로 인용** — CAV 임베딩 개입치다(E-C2).
4. 🚫 ARISE의 **"Function R@1 0.43→0.60(40%)·Line R@1 0.26→0.41(58%)"** — 이번 적대검증에서 **REFUTED (1-2)**. `+4.7pp`만 인용.
5. 🚫 **함수 라인 수 임계값**("N줄 초과 = 나쁨")을 점수·게이트로 사용 — 에이전트 정확도로 검증된 임계값 없음(cartography session-4 C8과 동일 판정).
6. 🚫 **LLM의 코드 이해 성공을 명명 품질의 오라클로 사용** — 부정확한 이름도 메워서 맞춘다(E-N4 false positive).
7. 🚫 **"테스트 통과 = 동작 보존"** 주장 — ≈21% 누락(E-A2), 구조 오라클은 13.3% 누락(E-S2).
8. 🚫 **주석 밀도 목표치**(예: "주석 비율 20%") — 최적점 미상(O-4).
9. 🚫 **0~100 단일 등급 산출** — 본문 층위 등급의 근거 없음. `ai-readiness-cartography`가 저장소 층 등급을 소유. 이 플러그인은 **census(인벤토리)** 만 낸다.
10. 🚫 **사람 가독성 포매팅**(들여쓰기·공백) 개선을 AI 가독성 근거로 제시 — LLM에 무익(cartography session-1 C3, arXiv:2508.13666).
11. 🚫 **공개 벤치마크(SWE-bench류) 점수로 개선 델타 주장** — 63%가 정답 회수(E-M1).
12. 🚫 **에이전트 자기보고로 동작 보존 판정** — 실행 관측만(E-A2·E-M2).
