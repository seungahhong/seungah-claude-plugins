# code-legibility-harness

> AI 에이전트가 읽기 좋은 코드를 만드는 일은 **주석을 더 다는 게 아니라 틀린 주석을 지우는 것**부터 시작한다.

코드 **본문** 층위(주석·독스트링 / 변수명·함수명·클래스·컴포넌트명 / 함수·모듈 granularity)를 다각도로 진단해 제안하고, **사용자 승인 후에만 단계별로 적용**한다.

## 왜 이 순서인가

"AI가 읽기 좋은 코드"라고 하면 대부분 *주석을 더 달고 큰 함수를 쪼개는* 것을 떠올린다. 2025~2026년 1차 근거는 반대를 말한다.

| 발견 | 근거 |
|------|------|
| **오도하는 주석 하나가 구조 변형 세 개를 합친 것과 맞먹는 해악**을 낸다(−13.47pp vs −14.04pp). Chain-of-Thought로도 방어되지 않는다. | CodeCrash, arXiv:2504.14119 (NeurIPS 2025) |
| **식별자의 의미 단서를 지우면 클래스 요약 정확도가 11~29pt 붕괴**한다. 구조만으로 풀려야 할 실행 예측조차 20pp 넘게 떨어진다. | arXiv:2510.03178 (2025-10) |
| **자동 생성 주석은 최상위 모델도 ~20%가 부정확**하고, 목적특화 파인튜닝 모델조차 35~45%가 틀린다. | arXiv:2406.14836(배경) · C4RLLaMA, ICSE 2025 |
| **LLM 리팩터링의 19~35%가 기능적으로 비등가**이고, **그중 ≈21%는 기존 테스트를 전부 통과**한다. | arXiv:2602.15761 (2026-02) |
| 자동 리팩터링의 주된 실패는 논리 오류가 아니라 **컴파일·참조 깨짐**이다. | arXiv:2605.22526 (2026-05) |
| 구조 층의 강한 수치(92.7%·+12%·+32.8%)는 전부 **코드를 파싱해 만든 툴 측 그래프 인덱스**의 효과이지 본문 리팩터의 효과가 아니다. | LocAgent 2503.09089 · RepoGraph 2410.14684 외 |

그래서 이 하네스는 **삭제를 추가보다 먼저 하고, 리네임을 구조보다 먼저 하며, 구조는 기본적으로 손대지 않는다.**

| | 개입 | 우선순위 | 왜 이 순위인가 |
|---|------|---------|---------------|
| **C0** | 오도·stale 주석 **삭제** | **P1** | 효과가 확실하고 **위험이 0**이다(주석은 실행되지 않는다) |
| **C1** | 계약·불변식 주석 **추가** | P3 | 효과는 있으나 **사실성 위험 20~45%** — 검증 게이트 필요 |
| **C2** | 무의미·오도 식별자 **리네임** | **P2** | 효과가 크되 참조를 깨뜨린다 — 도구가 하고 컴파일러가 검증 |
| **C3** | **구조 리팩터** | P4 · **기본 OFF** | **효과는 추론, 위험은 측정된 최대치** |

## 사용법

```
"AI가 읽기 좋게 주석이랑 변수명·함수명 정리해줘"
"코드 가독성 진단하고 승인하면 단계별로 적용해줘"
"주석이 코드랑 안 맞는 것 찾아서 고쳐줘"
"의미 없는 변수명 리네임 제안해줘"
```

### 흐름

```
Phase 0  초기 문의 ───── 대상 경로(스킵 가능) + 개입 클래스 체크박스(= 구속 스코프) + 리네임 도구 탐지
   │
Phase 1  census + 진단 ── legibility_scan.py (결정론) → 3 에이전트 병렬 (comment/naming/structure)
   │
Phase 2  제안표 ───────── 위험조정 랭킹 + 베이스라인 센서 실행       [게이트 A · 계획 승인]
   │
Phase 3  개별 적용 ────── 하나씩. 사전/사후 센서 실행·관측           [게이트 B · 개별 승인]
   │
Phase 4  재확인 ───────── census 델타 + 판정 집계                    [게이트 C · 최종 승인]
```

**승인 없이는 어떤 파일도 쓰지 않는다.** 커밋하지 않는다(`git-harness` 핸드오프).

### 결정론 스캐너 단독 실행

```bash
python3 skills/code-legibility-harness/scripts/legibility_scan.py <repo> \
  --axes naming,comments,structure --out census.json
```

stdlib만 쓴다(Python 3.10+). **코드를 수정하지 않는다.** 등급도 매기지 않는다 — census(인벤토리)만 낸다.

탐지기마다 근거 등급이 붙는다:

| 등급 | 뜻 | 개입 후보로 승격 |
|------|-----|----------------|
| `auto-high` | AST/토크나이저로 결정론 판정 | ✅ |
| `auto-med` | 결정론적 *후보*, 사람/LLM 확인 필요 | ✅ (확인 후) |
| `heuristic` | 근거가 약하거나 없음 | ❌ 진단 정보로만 |
| `report-only` | 판정하지 않음, 목록만 | ❌ |

| 탐지기 | 무엇을 잡나 |
|--------|-----------|
| `N1_NONDESCRIPTIVE` | `tmp`·`data`·`x2`·`Foo` 같은 정보량 0인 이름 |
| `N2_MISLEADING_PREDICATE` | `is_ready()`가 bool이 아닌 값을 반환 |
| `N3_MISLEADING_GETTER` | `get_total()`에 반환문이 없음 |
| `N4_SYMBOL_COLLISION` | 같은 이름이 여러 모듈에 (heuristic·근거 없음) |
| `C1_DANGLING_IDENT` | 주석이 존재하지 않는 식별자를 언급 |
| `C2_DOCSTRING_PARAM_MISMATCH` | docstring이 시그니처에 없는 파라미터를 문서화 |
| `C3_COMMENTED_OUT_CODE` | 주석처리된 코드 |
| `C4_DEBT_MARKER` | TODO/FIXME/HACK |
| `C6_COMMENT_DRIFT` | git blame 기준 주석-코드 시간 드리프트 (`--git-drift`) |

Python은 AST로 정밀 분석하고, 그 외 언어(TS/JS/Go/Rust/Java/…)는 주석·선언 수준의 보수적 분석(`auto-low`)을 한다.

## 정직성 — 이 하네스가 하지 않는 것

- **등급을 매기지 않는다.** 코드 본문 층위의 "몇 점"을 정당화할 1차 근거가 없다. 저장소 층 등급은 자매 플러그인 `ai-readiness-cartography`가 소유한다.
- **"테스트 통과 = 동작 보존"이라고 말하지 않는다.** 비등가 리팩터의 ≈21%가 테스트를 통과한다.
- **구조 개입에 수치를 약속하지 않는다.** 효과는 추론이다(미해결 질문 O-1).
- **LLM의 코드 이해를 명명 품질의 오라클로 쓰지 않는다.** 부정확한 이름도 모델이 메워서 맞춘다.
- **공개 벤치마크 점수로 개선을 주장하지 않는다.** SWE-bench Pro 성공의 63%가 정답 *회수*였다.
- **자기보고를 신뢰하지 않는다.** 센서를 실행해 관측한다.
- **리네임을 문자열 치환으로 하지 않는다.** AST/LSP 도구에 위임하고, 도구가 없으면 제안만 한다.

## 독립성

이 플러그인은 **단독으로 설치·실행된다.** 다른 플러그인이 없어도 모든 Phase가 동작한다.

- 스캐너는 **Python 표준 라이브러리만** 쓴다(3.10+). 외부 패키지도, 다른 플러그인의 스크립트도 import하지 않는다.
- 4개 에이전트 정의는 모두 이 플러그인 안에 있다.
- `git`은 `--git-drift`(선택 기능)에서만 쓰고, 없으면 그 탐지기만 건너뛴다.
- `git-harness`는 **선택적 동반 플러그인**이다. 이 스킬은 어떤 경우에도 스스로 커밋하지 않으므로, 없어도 지장이 없다.
- 안전 리네임 도구(LSP·rope·ts-morph 등)가 없으면 C2를 **"제안만"으로 강등**하고 계속 진행한다.

## 자매 플러그인과의 경계

아래 플러그인들은 **의존 대상이 아니라 스코프 구분 대상**이다(함께 쓰면 상보적이지만, 없어도 이 플러그인은 동작한다).

| 플러그인 | 층위 | 산출 |
|---------|------|------|
| `ai-readiness-cartography` | **저장소 층** — 문서·의존 그래프·CI·환경·게이트 | 100점 등급 + 대시보드. **코드를 수정하지 않는다.** |
| **`code-legibility-harness`** (이 플러그인) | **코드 본문 층** — 주석·식별자·함수/모듈 granularity | census + 개입 제안 + **승인 후 적용** |

겹치지 않는 이유: cartography는 **등급을 매기고 수정하지 않으며**, 이 플러그인은 **등급을 매기지 않고 수정한다**(승인 게이트 하에).

범위 밖: 세션 토큰 효율(`token-efficiency`), 테스트 계층 계획(`test-layering-harness`), 자가치유 QA(`qa-agent-harness`), 하네스 자체 진단(`meta-harness`), 실행 명세(`spec-driven-development`), 코드 리뷰·PR·커밋(`git-harness`).
