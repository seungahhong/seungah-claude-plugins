# doc-harness

## 하네스: 브랜치 기준 문서 자동 생성

**Goal:** 코드베이스와 현재 브랜치(git diff)를 기준으로 배경·목차·기술스펙·변경사항·테스트·영향을 포함한 기술 문서를 자동 생성하고 완성도를 검증한다.

**Trigger:** 브랜치/변경 기준 문서 생성·재생성·보완 요청 시 `branch-doc-orchestrator` 스킬을 사용한다. 단순 질의(예: "이 함수 뭐 하는 거야")는 직접 답변한다.

**실행 모드:** 서브에이전트 (수집 → 섹션 분석 병렬 팬아웃 → 조립 → 완성도 검증 루프). 모든 Agent 호출은 `model: "opus"`.
