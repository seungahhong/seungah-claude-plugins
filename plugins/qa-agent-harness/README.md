# qa-agent-harness

> 에이전트가 *커버리지를 채우는* 것이 아니라 **리스크 노출(발생확률 × 영향)을 커버리지 우선순위로 변환**하고, 각
> 시나리오에 **진짜 판정 오라클**을 명시한 뒤, 테스트를 생성·실행하되 실패를 맹목 재시도하지 않고 **결함/플래키/환경으로
> 먼저 분류**하는 도메인 무관(FE+API) 에이전틱 소프트웨어 QA 하네스.

근거: **arXiv:2601.02454 "The Rise of Agentic Testing" · arXiv:2506.02943 CANDOR · arXiv:2606.18168 "All Smoke, No Alarm" ·
arXiv:2504.16777 "Systemic Flakiness"**.

## 핵심 아이디어

**커버리지 극대화나 그린 신호 자체가 reward-hacking 가능한 대리지표다.** 실행 경로는 많이 밟지만 어서션이 약하면
*연기만 나고 경보는 울리지 않는다*(All Smoke, No Alarm). 그래서 이 하네스는 세 가지를 뒤집는다.

- **리스크가 노력을 배분한다** — 균일 커버리지가 아니라 발생확률×영향으로 무엇을 어느 깊이로 테스트할지 정한다.
- **오라클이 테스트의 가치다** — 테스트마다 *무엇이 성공/실패인가*(기대 상태·불변식)를 명시하고, 할루시네이션된 어서션은 다중 표본 합의로 거른다.
- **실패는 재시도 전에 분류한다** — 결함 vs 플래키 vs 환경으로 먼저 나누고, 공유 근본원인으로 클러스터링해 하나의 수정으로 다수 플래키를 해소한다.

전 과정은 **행동 전 제안(suggest-before-act) 점진적 자율성 + 파괴적/고위험 행동 HITL 게이트 + 설명 가능한 로깅된 결정**으로 관통된다.

## 언제 쓰나

- "이 기능의 **리스크를 기준으로** 테스트 전략을 세워줘"
- "이 사용자 스토리들을 **리스크에 매핑된 시나리오**로 설계하고 시나리오마다 **판정 오라클**을 붙여줘"
- "E2E 테스트를 생성해서 실행하고 깨진 로케이터는 **자가치유**(role/testid/접근성 트리 우선)하게 해줘"
- "테스트는 통과하는데 어서션이 미덥지 않아 — **오라클 품질을 감사**해줘"
- "CI에 실패 200개가 있는데 어느 게 **실제 결함이고 어느 게 플래키**인지 분류해줘"
- "이번 PR 변경분에 대해 **어떤 테스트를 먼저 돌릴지** 우선순위를 정해줘"
- "**행동 전 승인 게이트**를 두는 자율 QA 에이전트 파이프라인을 구성해줘"

## 4단계

| Phase | 이름 | 하는 일 | 게이트 |
|-------|------|---------|--------|
| 0 | 리스크 기반 테스트 전략 (Risk-Driven Test Strategy) | 요구사항·변경·리스크를 리스크 노출(확률×영향) 등급으로 정규화·커버리지 우선순위·판정 기준 | 승인 게이트 |
| 1 | 시나리오 설계·살아있는 라이브러리 (Scenario Design) | 스토리/API를 리스크 매핑 시나리오로 번역·시나리오마다 명시적 오라클(기대 상태·불변식) 부착 | 1줄 보고 |
| 2 | 오라클 우선 생성 + 자가치유 실행 (Generation + Self-Healing Execution) | 진짜 오라클 넣어 생성·실행·로케이터/DOM 드리프트 점수화·로깅된 자가치유 | 파괴적/고위험 HITL 게이트 |
| 3 | 실패 트리아지 & 근본원인 (Intelligent Failure Triage) | 결함/플래키/환경 분류·공유 근본원인 클러스터링·변경 기반 재실행 우선순위 | 트리아지 사람 게이트 |

> **자가치유**는 *맹목 재바인딩*이 아니다 — 점수화된 후보 비교로 치유하되 로깅·게이트되어, 실제 기능 변경/회귀를 초록으로
> 은폐하면 그 자체를 위반으로 잡는다. **트리아지**는 *그린까지 재시도*가 아니다 — 원인 분류 후에만 재실행한다.

## 사용법

스킬을 발동시키는 발화(위 "언제 쓰나")를 입력하면 오케스트레이터가 Phase 0부터 진행한다. Phase 0(리스크 전략) 직후
승인 게이트, Phase 2의 파괴적/고위험 행동·Phase 3의 최종 트리아지 판정에 사람 게이트가 있다.

## 언제 다른 도구를 쓰나 (도구 경계)

이 하네스는 **'리스크 기반 전략→오라클 명시 시나리오→오라클 우선 생성·게이트된 자가치유 실행→결함 vs 플래키 지능형
트리아지의 독립 end-to-end 에이전틱 QA 라이프사이클'** 에 특화한다. 다음은 범위 밖이다(일반 도메인 개념으로 서술 — 특정
플러그인에 의존하지 않는다).

- **넓은 프론트엔드 개발 워크플로우 안의 FE 한정 QA 단계** → FE 개발 도메인 (이 하네스는 도메인 무관 QA 라이프사이클로 자가치유·트리아지·리스크 전략을 1급으로 다룸)
- **구현 흐름 내부의 좁은 test-generator 스킬** → 백엔드 구현 도메인 (이 하네스는 특정 기능 구현이 아니라 전략→설계→실행→트리아지의 독립 방법론)
- **AI 출력을 오프라인 LLM-as-a-Judge로 채점·측정** → 평가 도메인 (이 하네스는 실제 테스트 스위트를 런타임에 생성·실행·자가치유·트리아지 — 측정이 아니라 실행·수리)
- **라이브 텔레메트리로 프로덕션 인시던트 RCA** → 운영 도메인 (이 하네스는 프리프로덕션 테스트 스위트 실패 트리아지)
- **코딩 착수 전 상류 인수조건↔테스트 커버리지 핸드오프 게이트 검수** → 핸드오프 리뷰 도메인 (이 하네스는 코드가 존재한 *후* 테스트를 실행·트리아지)
- **검증 가능한 목표로의 도메인 무관 자율 반복 루프** → 자율 반복 도메인 (정지조건 개념은 재사용 가능하나 이 하네스는 QA 특화 파이프라인)
- **여러 에이전트로 병렬화할지·토폴로지 결정** → 멀티 에이전트 오케스트레이션 도메인 / **사람↔에이전트 분업·감독 설계** → 사람-에이전트 협업 도메인
- **LLM 입출력 안전 가드레일(주입·PII·독성)** → 각 해당 도메인

## 근거 자료

설계는 Tech-Verse 2026 S06 "10x Speed With QA Agent Platform" 세션의 *하네스화 가능한 core*(조직 도입·성숙도 모델·변경관리
층은 제외)와 1차 자료 정독에 기반한다. 상세 인용·등급·CAVEAT는
[skills/qa-agent-harness/references/qa-agent-harness-research.md](skills/qa-agent-harness/references/qa-agent-harness-research.md) 참조.

- **[HIGH]** arXiv:2601.02454 "The Rise of Agentic Testing: Multi-Agent Systems for Robust Software Quality Assurance" — 에이전틱 QA 멀티에이전트 프레임.
- **[HIGH]** arXiv:2506.02943 CANDOR "Hallucination to Consensus" — 다중 에이전트 합의로 정확한 오라클 생성.
- **[HIGH]** arXiv:2606.18168 "All Smoke, No Alarm: Oracle Signals in Agent-Authored Test Code" — 커버리지≠결함 탐지, 오라클 신호의 load-bearing 근거.
- **[HIGH]** arXiv:2504.16777 "Systemic Flakiness" — 플래키의 상당수가 공동발생 클러스터, 공유 근본원인.
- **[MEDIUM]** arXiv:2601.05542 LLM 오라클 생성 이해 · Requirements-based test prioritization(산업연구, 리스크 인자) · Katalon Agentic QA 운영모델.

> **정직성**: S06 세션의 헤드라인 신규성인 *조직 도입·AI 성숙도 5단계·변경관리* 층은 Claude Code로 따를 수 없어 명시적으로
> 제외한다. 회사별 수치(1.54x QA 산출·~49% AI 대체·68% 시간 절감)와 CANDOR 오라클 정확도(0.834→0.971)는 *특정 세팅
> 관찰값*이지 약속이 아니며 "개선 N% 보장"을 쓰지 않는다.

## 독립성

다른 마켓플레이스 플러그인에 의존하지 않는 **독립 플러그인**이다. 경계의 '범위 밖'은 일반 도메인 개념으로만 서술하며,
모든 근거는 1차 자료를 직접 인용한다.
