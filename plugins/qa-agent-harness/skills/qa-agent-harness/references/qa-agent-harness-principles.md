# QA Agent Harness — 원칙·anti-pattern·결정 신호표

> `qa-agent-harness` 하네스의 설계 헌법. 모든 Phase가 따른다. 근거는 [qa-agent-harness-research.md](./qa-agent-harness-research.md)
> (§N은 그 dossier의 절). 핵심 전제: **커버리지 극대화나 그린 신호 자체가 reward-hacking 가능한 대리지표다 — 오라클이
> 약하면 연기만 나고 경보는 울리지 않는다**. 본 하네스는 이 기반 위에서 **리스크→오라클→실행→트리아지의 거버넌스된
> 에이전틱 QA 라이프사이클 한 번**을 운영한다(research §1·§3).

---

## §1. 리스크 노출이 노력을 배분한다 (risk-driven effort allocation)

QA의 노력은 *균일 커버리지*가 아니라 **리스크 노출(발생확률 × 영향)** 이 배분한다(research §6 요구사항 기반 리스크
우선순위화 산업연구). 무엇을 어느 깊이·순서로 테스트할지는 리스크 노출도가 정한다. 변경 영향·과거 결함·외부 의존이 리스크
등급의 근거다.

## §2. 오라클 우선 (oracle-first) — *커버리지 ≠ 결함 탐지*

테스트의 가치는 실행 경로 수가 아니라 **어서션(오라클)의 강도**에 있다(research §3 "All Smoke, No Alarm"). 커버리지는
결함 탐지력과 *약하게만* 상관하므로 **스모크/동어반복 오라클**(예: 200 떴으니 통과, 요소 존재만 확인)을 금지하고 시나리오
마다 *기대 상태·불변식*을 명시한다. 라이브러리는 정적 회귀팩이 아니라 제품과 함께 진화하는 *살아있는 라이브러리*다.

## §3. 오라클을 독립 검증한다 (consensus over hallucination)

할루시네이션된 어서션은 *그럴듯하지만 의미적으로 틀려* 정답 프로그램에서 실패하거나 버그를 통과시킨다. 이런 오라클은
모델 간 *비일관적*이므로 **다중 표본 합의(panel/consensus)** 로 걸러낸다(research §2 CANDOR). CANDOR가 보고한 오라클
정확도 0.834→0.971은 *특정 세팅 관찰값*이지 보편 법칙·약속이 아니다(§10).

## §4. 자가치유는 점수화·로깅·게이트 (scored / logged / gated healing)

로케이터/DOM 드리프트는 *맹목 재바인딩*이 아니라 **점수화된 후보 비교**로 치유한다 — 취약한 XPath/CSS 대신
*role → data-testid → 접근성 트리 → 가시 텍스트* 우선순위로 후보를 비교한다(research §1·§7 자가치유). 모든 치유를 *로깅*하고
신뢰도 낮은 치유·기능 변경 의심은 *사람 검토로 게이트*한다. **실제 기능 변경/회귀를 초록으로 은폐하는 재바인딩은 위반이다.**

## §5. 재시도 전 트리아지 (triage before retry)

실패는 재시도하기 전에 **실제 결함 vs 플래키 vs 환경**으로 분류한다(research §1·§4). 플래키는 *공유 근본원인*(네트워킹·외부
의존 등)으로 클러스터링해 하나의 수정으로 다수를 해소한다 — 플래키의 상당수가 *공동발생 클러스터*이므로 체계적(systemic)
플래키를 고립 사건으로 오판하지 않는다(research §4 Systemic Flakiness). *실제 결함을 플래키로 오분류해 삼키지 않는다.*

## §6. 변경 기반 우선순위화 (change-based prioritization)

회귀 전량 재실행이 아니라 **코드 변경 영향 + 리스크**로 테스트를 선택·정렬한다(research §6 영향 기반 선택 — 유출 증가
없이 노력 절감, 세팅 관찰값). 무엇을 먼저·왜 돌리는지가 설명 가능해야 한다.

## §7. 점진적 자율성 + HITL 게이트 (progressive autonomy, suggest-before-act)

**행동 전 제안(suggest-before-act)**, 신뢰가 쌓이며 자율 범위를 넓히되 **파괴적/고위험 행동**(테스트 데이터 대량 삭제·외부
시스템 변조·프로덕션 접촉)은 *사람 승인 게이트*를 통과시킨다(research §7 운영모델). 가역·저위험은 진행, 게이트를 *필요한
곳*에 집중해 마찰을 줄이되 비가역 손상을 막는다.

## §8. 설명 가능한 로깅된 결정 (explainable, logged decisions)

전략 우선순위·치유 선택·트리아지 판정을 *근거와 함께* 기록해 감사·재현 가능하게 한다(research §1·§7). 설명 없는 자동
결정은 감사 불가능하고 신뢰를 무너뜨린다.

## §9. 커버리지 연극·리워드 해킹 가드 (reward-hacking guard)

그린 신호는 *대리지표*다 — 최적화 대상으로 삼으면 게이밍된다(research §3). **위반으로 잡는다**: 실행 경로만 밟고 어서션이
약한 *커버리지 연극(All Smoke, No Alarm)*, 자가치유가 회귀를 초록으로 은폐, *그린까지 재시도/플래키 억제*, 현재 구현
동작에 오버피팅한 오라클(화이트박스 통과·홀드아웃 블랙박스 실패)이다. 센서(오라클)가 약하면 *보강하거나 격차를 명시*하지,
약한 통과를 결함 없음으로 단정하지 않는다.

## §10. 정직성·falsifiability

- Tech-Verse 2026 S06 세션의 헤드라인 신규성인 **조직 도입·AI 성숙도 5단계·변경관리 층**은 Claude Code로 따를 수 없어
  *명시적으로 제외*한다(research 제외 범위). 본 하네스는 *하네스화 가능한 에이전틱 QA 파이프라인*만 담는다.
- 회사별 수치(**1.54x QA 산출·~49% AI 대체·68% 시간 절감**)와 CANDOR 오라클 정확도(**0.834→0.971**), 영향 기반 선택의
  절감은 *특정 세팅·코호트 관찰값*이지 약속이 아니다. **"개선 N% 보장"을 쓰지 않는다.**
- 근거는 *1차 자료를 직접 인용*하고 등급(HIGH/MEDIUM)을 붙인다. 벤더 자료(Katalon)는 *운영모델 어휘*로만 쓰고 성과 주장을 인용하지 않는다.

## §11. 승인 게이트·관찰성

Phase 0(리스크 전략) 직후 승인 게이트는 항상. Phase 2의 파괴적/고위험 행동·Phase 3의 최종 트리아지 판정도 사람 게이트.
각 Phase는 1줄로 보고하고, *요청되지 않은 사이드 에이전트나 중복 실행*을 만들지 않는다.

## Anti-pattern (하지 말 것)

| Anti-pattern | 왜 나쁜가 | 올바른 패턴 | 근거 |
|---|---|---|---|
| 커버리지 연극(All Smoke, No Alarm) | 경로만 밟고 어서션 약함 — 결함 못 잡음 | 오라클 우선(기대 상태·불변식 명시) | §2·§9·research §3 |
| 할루시네이션된 오라클 | 정답에서 실패·버그를 통과시킴 | 다중 표본 합의로 검증 | §3·research §2 |
| 균일 커버리지 목표화 | 리스크 무시·대리지표 게이밍 | 리스크 노출로 노력 배분 | §1·§9·research §6 |
| 자가치유가 회귀를 은폐 | 진짜 결함을 초록으로 삼킴 | 점수화·로깅·게이트된 치유 | §4·research §1 |
| 그린까지 재시도/플래키 억제 | 근본원인 은폐·systemic 오판 | 재시도 전 트리아지·클러스터링 | §5·§9·research §4 |
| 테스트 오버피팅(현재 구현에 맞춤) | 명세 위반 못 잡음·리워드 해킹 | 명세 기반 오라클 | §2·§9·research §5 |
| 회귀 전량 재실행 | 비용·느림 | 변경 영향+리스크 우선순위화 | §6·research §6 |
| 파괴적/고위험 행동 자율 실행 | 비가역 손상 | HITL 게이트(suggest-before-act) | §7·research §7 |
| 설명 없는 자동 결정 | 감사·재현 불가 | 근거와 함께 로깅 | §8·research §1 |
| 조직 도입·성숙도 모델을 하네스에 넣기 | 하네스화 불가·과장 | 하네스화 가능 core만 | §10·research 제외 범위 |
