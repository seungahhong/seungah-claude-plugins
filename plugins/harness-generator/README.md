# harness-generator

특정 도메인에 종속되지 않는 멀티 에이전트 하네스(에이전트 팀 + 스킬 + 오케스트레이터)를 자동으로 설계·생성하는 Claude Code 메타 플러그인입니다. 새로운 도메인의 다단계 워크플로우(연구 → 설계 → 실행 → 검증)를 체계적으로 자동화하고 싶을 때 사용합니다.

## 설치

```bash
claude plugin add seungahhong/seungah-claude-plugins
```

## 스킬

| Skill | Command | 설명 |
|-------|---------|------|
| Harness Generator | `/harness-generator` | 7단계 메타 프로세스로 도메인용 하네스를 한 묶음으로 설계·생성·검증 |

## 언제 쓰나

- 단일 스킬/일회성 프롬프트로 처리하기 어려운 **다단계 작업**(분석 → 설계 → 실행 → 검증)을 자동화할 때
- 여러 **에이전트/스킬/오케스트레이터를 한 묶음**으로 설계해야 할 때
- 기존 하네스를 **확장·보완·감사**할 때

단순한 단일 스킬 추가에는 마켓플레이스 공식 `skill-creator`가 더 적합합니다.

## 사용법

### 기본 호출

만들고 싶은 도메인/워크플로우를 함께 설명하며 호출합니다.

```
/harness-generator
```

예시 입력:

> "PR 리뷰부터 배포까지 자동화하는 하네스를 만들어줘. 리뷰 → 승인 → 배포 → 검증 단계로."

또는 자연어로: "하네스 만들어줘", "오케스트레이터 작성해줘", "에이전트 팀 구성해줘", "이 도메인용 스킬 묶음 생성해줘".

### 생성 과정 — 7단계

| Phase | 단계 | 하는 일 |
|-------|------|---------|
| 0 | 현황 감사 | 신규 도메인 / 기존 하네스 확장 / 단순 보완을 자동 판별해 분기 |
| 1 | 도메인 분석 | 입력·산출물·필요한 단계를 정의 |
| 2 | 아키텍처 설계 | 실행 모드(팀 / 서브에이전트 / 하이브리드) + 아키텍처 패턴(파이프라인 등) 선택, 에이전트 분리 기준 4축 적용 |
| 3 | 에이전트 정의 작성 | 역할·원칙·입출력·에러 처리·협업 프로토콜을 갖춘 `agents/{name}.md` 생성 |
| 4 | 스킬 작성 | 단계별 `skills/{step}/SKILL.md` + 필요 시 출력 데이터 스키마 |
| 5 | 오케스트레이션 | 흐름을 통제하는 진입점 오케스트레이터 작성 (`_workspace/` 컨텍스트 관리 포함) |
| 6 | 검증 | 구조 검증·모드별 점검·스킬 실행 테스트 |

### 재실행·보완

이미 만든 하네스를 다시 호출하면 Phase 0에서 기존 산출물을 감지해 **확장/부분 재실행/단순 보완**으로 분기합니다. "이 하네스에 검증 단계 추가해줘"처럼 요청하면 됩니다.

## 산출물

대상 위치에 아래 구조로 하네스 한 묶음이 생성됩니다.

```
{harness-root}/
  agents/{agent}.md            # 에이전트 정의
  skills/{step}/SKILL.md       # 단계별 스킬
  (오케스트레이터 스킬/커맨드)   # 전체 흐름 진입점
```
